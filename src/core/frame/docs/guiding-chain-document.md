# 导引链（Guiding-chain）设计文档

## 1. 概述

导引链用于串联取景框或取景框模板，形成一条有始有终的演示路径。通过导引链，用户可以定义镜头在画布上的移动顺序，实现平滑的演示导航。

## 2. 术语定义

| 术语 | 定义 |
|------|------|
| **导引链** | 串联取景框或模板实例的线性演示路径 |
| **节点** | 导引链中的单个元素，可以是取景框实例或模板实例 |
| **父链** | 位于画布顶层的导引链，串联取景框和模板实例 |
| **子导引链** | 定义在模板实例内部的导引链，仅串联该模板内部取景框 |
| **镜头** | 用户当前可视区域的抽象，沿导引链移动以切换视图 |
| **黑盒节点** | 指导引链中的模板实例节点，外部不感知其内部结构和子链 |

## 3. 核心定位

### 3.1 本质

导引链是**定义镜头移动顺序的逻辑结构**，不直接操控镜头，而是提供一串有序的“目标取景框”。镜头设备体系据此执行平滑移动和变焦。

### 3.2 用途

- 演示模式下，按预设顺序依次聚焦取景框或模板
- 为画布内容提供结构化的浏览路径
- 通过子导引链，在模板内部组织局部演示

### 3.3 与取景框/模板的关系

- 导引链引用取景框或模板实例作为节点，不拥有它们
- 取景框或模板实例可被多条导引链引用，系统会提示未被任何链引用的对象
- 模板实例对父链表现为黑盒节点，父链只负责将镜头带到模板，模板内部的导航由模板自身的移动控件或子导引链负责

## 4. 链结构

### 4.1 拓扑

- 导引链为**严格线性结构**，有明确的起始节点和终止节点
- 不支持分支（从一个节点分叉到多个后续节点）
- 不支持回路（最后一个节点不能指向第一个节点）

### 4.2 节点类型

| 节点类型 | 说明 |
|----------|------|
| **取景框实例** | 画布上的普通取景框 |
| **模板实例** | 取景框模板的具体实例，作为黑盒节点 |

### 4.3 节点顺序

- 节点按用户定义的顺序排列
- 镜头沿顺序从前到后移动，到达终止节点后演示结束

## 5. 多条导引链

### 5.1 并存规则

- 画布上可同时存在多条导引链
- 多条链之间互不干扰，各自独立
- 每条链拥有自己的节点序列和播放状态

### 5.2 未引用提醒

- 系统会识别未被任何导引链引用的取景框或模板实例
- 以适当方式提醒用户存在未纳入演示路径的对象

## 6. 子导引链

### 6.1 定义与归属

- 子导引链定义在**模板实例内部**
- 属于模板实例级别，需手动创建
- 子导引链的节点只能是**该模板实例内部的取景框**

### 6.2 约束

- **外部禁止**：子导引链不能引用模板外部的任何取景框
- **嵌套禁止**：子导引链内部不允许再创建子导引链（仅一层）
- **黑盒边界**：父链不感知模板内部是否存在子导引链，子导引链在模板内部独立运作

### 6.3 用途

在模板内部的多个取景框之间定义局部演示路径，使模板内的分页内容能够按特定顺序浏览。

## 7. 镜头行为

### 7.1 移动方式

- 当演示沿导引链推进时，镜头从一个节点平滑移动到下一个节点
- 移动过程包含平移和变焦，以自然适配目标取景框的区域
- 具体过渡效果由镜头设备体系实现，本文不展开

### 7.2 模板节点处理

- 当镜头到达模板实例节点时，控制权移交给模板
- 模板内部由模板自身的移动控件或子导引链接管镜头
- 父链不感知模板内部的镜头移动细节

### 7.3 手动干预

- 演示过程中，用户可手动移动镜头，甚至吸附到其他节点进行查看
- 手动移动不影响导引链的播放进程
- 导引链的当前节点位置不会因手动浏览而改变

## 8. 与其他模块的关系

| 模块 | 关系 |
|------|------|
| **取景框** | 取景框可作为导引链节点；系统识别未引用取景框 |
| **取景框模板** | 模板实例作为黑盒节点；模板内部可包含子导引链 |
| **镜头/设备体系** | 导引链提供有序的目标取景框列表，镜头设备执行移动 |
| **元素系统** | 无直接关联，导引链不操作元素 |
