
# 取景框（Frame）设计文档

## 1. 概述

取景框是无限画布上的矩形区域，核心用途是为镜头提供对齐目标。它同时可兼作导出裁剪区域、演示幻灯片页面、内容分组容器和画布边界标识。

## 2. 术语定义

| 术语 | 定义 |
|------|------|
| **取景框** | 画布上的矩形区域，用于定义镜头对齐目标 |
| **白板对象** | 画布上所有可编辑实体的统称，包括元素、取景框等 |
| **锚点** | 元素上用于判断其与取景框包含关系的参考点 |
| **镜头** | 用户当前可视区域的抽象，可对齐到取景框 |
| **层叠图** | 已有系统，管理白板对象的层级和绘制顺序 |

## 3. 核心定位

### 3.1 本质

取景框是**独立的白板对象**。它不拥有、不绑定任何元素，与元素之间不存在直接的"归属"关系。

### 3.2 用途

- 导出/打印时的裁剪区域
- 演示模式下的幻灯片页面
- 画布上的内容分组容器
- 无限画布中有限区域的边界标识

### 3.3 形状与旋转

- 取景框的形状固定为**矩形**
- 支持**任意角度旋转**

## 4. 与元素的关系

### 4.1 包含判断

元素是否被取景框"包含"，由以下两种方式之一判定：

- **锚点判断（默认）**：元素的锚点落在取景框矩形范围内，即视为包含
- **元素重叠判断**：基于元素可视范围与取景框的重叠情况进行判定，细分为：
  - **完全包含**：元素整体在取景框内
  - **沾边**：元素与取景框有任何重叠即算包含

包含关系用于镜头对齐、导出裁剪等场景中确定"框内有什么"，但不建立绑定关系。

### 4.2 边界行为

- 元素可以跨越取景框边界显示，取景框不裁剪元素
- 取景框不影响元素的编辑、移动、删除等操作

## 5. 取景框作为白板对象

### 5.1 层级

取景框无层级体系，与普通白板对象独立。

### 5.2 选择行为

取景框的选择与普通白板对象**无本质区别**，但取景框和普通白板对象的选择使用不同的选择管线。

### 5.3 编辑操作

| 操作 | 行为 |
|------|------|
| **移动** | 与普通对象一致 |
| **缩放** | 独立调整大小，不影响框内元素 |
| **旋转** | 支持任意角度旋转 |
| **锁定** | 支持锁定（位置、尺寸不可编辑） |

### 5.4 删除与解散

- **删除取景框及内容**：不仅移除取景框本身，也移除范围内画布上的元素
- **解散取景框**：移除取景框，元素保留。作为显式功能提供，语义上强调"保留内容"

## 6. 镜头对齐

### 6.1 基本行为

用户可将镜头对齐到取景框，镜头自动平移和缩放以适配取景框区域。这是取景框的核心用途之一。

### 6.2 与导引链的关系

取景框可作为导引链的节点。导引链串联多个取景框形成演示路径。具体镜头移动由导引链和设备体系控制，本文不展开。

## 7. 导航

取景框本身不提供独立的缩略图导航面板。取景框之间的导航和跳转由**导引链**承担。

## 8. 与其他模块的关系

| 模块 | 关系 |
|------|------|
| **取景框模板** | 模板可生成附属性的取景框集合，与普通取景框不同（详见模板文档） |
| **导引链** | 取景框可作为导引链节点，被串联进演示路径 |
| **元素系统** | 通过包含判断产生弱关联，无直接绑定 |
| **层叠图** | 取景框不参与层级排列 |
| **镜头/设备体系** | 镜头可对齐到取景框，由设备体系执行平移和变焦 |
